# GitHub Profile Plan

GitHub 프로필 첫 화면은 `Signal Archive`를 중심 허브로 두고, 개별 프로젝트는 역할 적합성을 보여주는 증거로 배치합니다.

## Pinned Repositories

추천 순서:

1. `signal-archive`
2. `Seoul-Storefront-Redveil`
3. `LH-traffic-safety-analysis`
4. `starbucks-promotion-analysis`
5. `UK-OnlineRetail-Segment-analysis`
6. `ShopEasy`

선택 이유:

- `signal-archive`는 전체 포트폴리오 허브이므로 가장 먼저 보여주는 것이 자연스럽습니다.
- `Seoul-Storefront-Redveil`은 공개 웹 서비스가 있어 첫 화면 체감이 좋습니다.
- `LH-traffic-safety-analysis`는 공공데이터와 공간 분석 역량을 보여줍니다.
- `starbucks-promotion-analysis`와 `UK-OnlineRetail-Segment-analysis`는 CRM/마케팅/커머스 분석 축을 보강합니다.
- `ShopEasy`는 대시보드형 결과물 전달 역량을 보여줍니다.

보류 후보:

- `nba-game-player-analysis`는 스포츠 분석 역량을 보여주지만, 현재 포지셔닝에서는 공공/상권/CRM 축보다 우선순위가 낮습니다.

## Profile README Message

핵심 메시지:

- 문제를 다시 정의하고 데이터를 판단 가능한 구조로 바꾸는 데이터 분석가
- 공공데이터, 도시/상권, CRM, 리테일 데이터를 다루는 분석 포트폴리오
- 분석 결과를 문서, 대시보드, 웹 결과물로 끝까지 연결하는 실행력

피해야 할 표현:

- 도구 이름만 나열하는 자기소개

- 너무 추상적인 성실성, 열정, 성장성 강조

## Publish Notes

- GitHub 프로필 README는 사용자명과 동일한 public repository인 `dffxonnb-cyber/dffxonnb-cyber`가 필요합니다.
- 현재 로컬 폴더는 git repo이지만 원격 저장소가 아직 없으면 push가 실패합니다.
- 원격 저장소 생성 후 `git remote add origin https://github.com/dffxonnb-cyber/dffxonnb-cyber.git`와 `git push -u origin main`을 실행합니다.
